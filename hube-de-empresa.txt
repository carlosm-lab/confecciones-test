## 1. PROPÓSITO DE LA PÁGINA

Esta página es el Hub central de la sección `/empresa` del sitio web corporativo de Confecciones Liss. Su función no es decorativa. Su función es estratégica: actúa como puerta de entrada institucional, presenta la empresa con autoridad, genera confianza inmediata y distribuye al visitante hacia las secciones internas mediante un sistema de tarjetas de navegación claramente jerarquizado.

El visitante que llega a `/empresa` debe comprender en menos de 5 segundos que está frente a una empresa familiar salvadoreña seria, con historia verificable, equipo real, procesos definidos y presencia digital propia. Debe sentir confianza antes de hacer clic en cualquier sección.

Esta página no vende. Esta página establece autoridad y credibilidad.

### Sistema de grid de las tarjetas

Las 11 tarjetas se distribuyen en un layout **Bento asimétrico**. No es un grid uniforme. Hay tarjetas de diferentes tamaños para crear jerarquía visual.

**Desktop (≥1024px):**
```
FILA 1: [TARJETA A — grande 2/3] [TARJETA B — mediana 1/3]
FILA 2: [TARJETA C — mediana 1/3] [TARJETA D — mediana 1/3] [TARJETA E — mediana 1/3]
FILA 3: [TARJETA F — grande 1/2] [TARJETA G — mediana 1/4] [TARJETA H — mediana 1/4]
FILA 4: [TARJETA I — mediana 1/3] [TARJETA J — mediana 1/3] [TARJETA K — mediana 1/3]
```

Implementar con CSS Grid:
```css
grid-template-columns: repeat(6, 1fr);
gap: 16px;

Tarjeta A: col-span-4
Tarjeta B: col-span-2
Tarjeta C: col-span-2
Tarjeta D: col-span-2
Tarjeta E: col-span-2
Tarjeta F: col-span-3
Tarjeta G: col-span-1 (ajustar si no cuadra → col-span-1.5 con subgrid o adaptar a col-span-2)
Tarjeta H: col-span-1 (igual)
Tarjeta I: col-span-2
Tarjeta J: col-span-2
Tarjeta K: col-span-2
```

Nota: Si el subgrid genera conflictos, adaptar a:
```
FILA 3: [TARJETA F — col-span-3] [TARJETA G — col-span-1 y medio aprox] [TARJETA H — col-span-1 y medio aprox]
```
→ Usar `grid-template-columns: repeat(6, 1fr)` y asignar F=3, G=2, H=1 o F=3, G=1, H=2 dependiendo de cuál va primero. El desarrollador debe mantener la proporción visual: F es la más grande de la fila.

**Tablet (768–1023px):**
Todas las tarjetas en grid de 2 columnas iguales. La Tarjeta A ocupa las 2 columnas (col-span-2).

**Mobile (<768px):**
Todas las tarjetas en 1 columna. Todas iguales en ancho.

### Altura mínima de las tarjetas por tamaño:
- Grande (col-span-4 / col-span-3): min-height 280px desktop

- Pequeña (col-span-1): min-height 200px desktop

## 13. ANIMACIONES Y MICROINTERACCIONES

### Entrada de la página
Usar `framer-motion` con `AnimatePresence`. La página entera hace fade-in suave en 300ms con `opacity: 0 → 1`. Sin movimiento vertical en el fade-in de la página.

### Entradas por scroll (IntersectionObserver o Framer Motion `whileInView`)
Aplicar a: Bloque 3, Bloque 4, Bloque 5, Bloque 6, Bloque 7.

```
Variante de entrada estándar:
  initial:   { opacity: 0, y: 24 }
  animate:   { opacity: 1, y: 0 }
  transition: { duration: 0.5, ease: [0.25, 0.46, 0.45, 0.94] }
  once:      true (no repetir al hacer scroll de vuelta)
```

Las tarjetas del Bloque 4 entran con un **stagger** de 60ms entre cada tarjeta. El orden del stagger sigue el orden visual del grid (izquierda a derecha, arriba a abajo).

```
Stagger:
  transition: { delay: index * 0.06 }
```

### Hover en tarjetas
- Transform: `translateY(-3px)`, duration 260ms
- Box-shadow: cambio de SHADOW_SM a SHADOW_HOVER, duration 260ms
- Ícono ArrowUpRight: `opacity: 0 → 1`, `transform: translateX(-4px) translateY(4px) → translateX(0) translateY(0)`, duration 200ms

### Respeto a preferencias del sistema
Implementar `@media (prefers-reduced-motion: reduce)` deshabilitando transforms y animaciones. Solo permitir opacity transitions.

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```
